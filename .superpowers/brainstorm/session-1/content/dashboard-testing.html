Dashboards Design — Section 4: Testing

Unit, integration, and authorization test strategy for Gruuber.Analytics.

Unit Tests (Moq)

AnalyticsConsumerService
 · ride_completed event → correct delta upserted to driver_stats_daily and admin_stats_daily
 · order_delivered event → correct upsert across restaurant + menu item + admin tables
 · Duplicate event (same event_id) → second upsert is skipped (idempotency)
 · Rolling average rating recalculation is correct

Query Handlers
 · Weekly summary correctly sums 7 daily rows
 · Monthly summary correctly sums up to 31 daily rows
 · No data for period → returns zero-valued summary (not null/404)

Export Service
 · CSV output matches expected columns and row count
 · Job status transitions: pending → processing → completed / failed

Integration Tests (Testcontainers — Postgres + Kafka)

 · Publish 5 ride_completed events → assert driver_stats_daily has correct totals
 · Publish order_delivered with 3 items → assert menu_item_stats_daily has 3 rows
 · Duplicate event replay → totals unchanged (dedup working)
 · Consumer failure after 5 retries → message in analytics-events-dlq-{region}
 · Export job: request CSV → poll → download URL returns valid file

Authorization Tests

 · Driver A cannot access Driver B's summary (JWT sub mismatch → 403)
 · Restaurant cannot access driver endpoints (403)
 · Admin can query any region; driver/restaurant cannot use admin endpoints (403)
 · Export download URL with expired TTL → 403 / 410 Gone
