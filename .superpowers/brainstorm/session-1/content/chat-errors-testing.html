In-App Chat Design — Section 3: Error Handling, Observability & Testing

Failure modes, logging, metrics, and test strategy for Gruuber.Chat.

Error Handling

Scenario Handling
Message sent to read_only/closed thread SignalR error response to sender: THREAD_CLOSED
User not in chat_participants tries to join Hub rejects connection with 403 Forbidden
Kafka lag — thread not yet created when client connects Client retries GET /v1/chat/threads?context_id= with backoff (max 5s)
SignalR client disconnects mid-conversation Messages queued in DB; delivered as 'delivered' on reconnect via history endpoint
Kafka consumer fails to create thread Retry with backoff (max 5); route to chat-events-dlq-{region} on failure
Quick reply key not found 400 Bad Request — invalid template key

Observability

Log fields
 TraceId, SpanId, RegionId, ThreadId, ContextId, ContextType, Channel

Metric Purpose
chat_messages_per_trip Avg messages sent per active thread — engagement signal
chat_thread_creation_lag_ms Time from Kafka event → thread available — alert if > 2s
chat_signalr_connections Active ChatHub connections gauge
chat_dlq_events Failed thread creation events routed to DLQ

Testing

Unit Tests (Moq)
 · Thread creation from ride_matched event → correct participants and channel
 · Thread creation from order_accepted → two threads created (customer↔restaurant + driver↔restaurant)
 · Message to read_only thread → rejected with THREAD_CLOSED
 · MarkRead → status updated, read_at set, sender notified
 · Quick reply with invalid key → 400 Bad Request

Integration Tests (Testcontainers — Postgres + Kafka + SignalR)
 · End-to-end: ride_matched event → thread created → rider sends message → driver receives via SignalR
 · Read receipt: driver reads → sender gets double-tick notification
 · Thread closes 6h after completion → message send rejected
 · Client reconnect → history endpoint returns missed messages in order
 · Consumer fails 5 times → event in chat-events-dlq-{region}

Privacy Invariant
 · Assert: chat_participants.display_name never contains real name, phone, or email
 · Assert: user not in chat_participants cannot join or read thread messages
