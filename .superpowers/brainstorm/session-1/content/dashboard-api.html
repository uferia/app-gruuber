Dashboards Design — Section 2: API & Kafka Consumers

Endpoints for each role + the Kafka consumer upsert pattern that populates the tables.

Driver Dashboard API

GET /v1/analytics/driver/summary?period=daily|weekly|monthly
  → earnings, trips, rating, acceptance_rate, online_time, bonuses, payouts

GET /v1/analytics/driver/trips?page=1&limit=20&from=&to=
  → paginated trip history with fare, status, pool_slot

GET /v1/analytics/driver/earnings/export?format=csv|pdf&from=&to=
  → 202 Accepted + job_id (async export generation)

GET /v1/analytics/driver/exports/{job_id}
  → 200 { download_url } | 202 { status: "processing" }

Restaurant Dashboard API

GET /v1/analytics/restaurant/summary?period=daily|weekly|monthly
  → orders, revenue, avg_prep_time, avg_rating, cancellation_rate

GET /v1/analytics/restaurant/orders?page=1&limit=20&from=&to=
  → paginated order history

GET /v1/analytics/restaurant/menu-performance?period=weekly|monthly
  → items ranked by units_sold + revenue (best/worst sellers)

GET /v1/analytics/restaurant/revenue/export?format=csv|pdf&from=&to=
  → async export, same job_id pattern as driver

Admin Dashboard API

GET /v1/analytics/admin/summary?region_id=&period=daily|weekly|monthly
  → platform totals: rides, orders, revenue, active drivers/restaurants

GET /v1/analytics/admin/export?format=csv|pdf&region_id=&from=&to=
  → async export

Kafka Consumer Upsert Pattern

AnalyticsConsumerService — listens to all region topics
On ride_completed → UPSERT driver_stats_daily (driver_id, TODAY)
                  UPSERT admin_stats_daily (region_id, TODAY)

On order_delivered → UPSERT restaurant_stats_daily (restaurant_id, TODAY)
                   UPSERT menu_item_stats_daily per order_item
                   UPSERT admin_stats_daily (region_id, TODAY)

On ride_rated / order_rated → recalc avg_rating via rolling average upsert

All upserts use INSERT ... ON CONFLICT (pk) DO UPDATE SET col = col + delta
Idempotency: each event carries event_id; consumer deduplicates via processed_events table
