Surge Pricing Design — Section 1: Data Model

Configuration tables, ride/order schema additions, and Redis keys.

New Table: surge_config

CREATE TABLE surge_config (
  region_id        INT NOT NULL,
  ride_type        TEXT NOT NULL, -- 'ride' | 'food'
  demand_ratio_threshold NUMERIC NOT NULL, -- e.g. 0.7 = 70% drivers busy
  multiplier       NUMERIC NOT NULL, -- e.g. 1.5
  max_multiplier   NUMERIC NOT NULL, -- hard cap, e.g. 3.0
  updated_at       TIMESTAMP,
  PRIMARY KEY (region_id, ride_type, demand_ratio_threshold)
);

-- Multiple rows per region/type define the stepped tiers:
-- threshold 0.5 → 1.5x, threshold 0.75 → 2.0x, threshold 0.90 → 2.5x

New Table: surge_time_rules

CREATE TABLE surge_time_rules (
  id         UUID PRIMARY KEY,
  region_id   INT NOT NULL,
  ride_type   TEXT NOT NULL,
  day_of_week INT NOT NULL, -- 0=Sun … 6=Sat, NULL = every day
  start_time  TIME NOT NULL,
  end_time    TIME NOT NULL,
  multiplier  NUMERIC NOT NULL, -- applied if active, overrides demand tier
  is_active   BOOLEAN NOT NULL DEFAULT true
);

Schema Additions to rides and orders

ALTER TABLE rides ADD COLUMN base_fare      NUMERIC;
ALTER TABLE rides ADD COLUMN surge_multiplier NUMERIC NOT NULL DEFAULT 1.0;
ALTER TABLE rides ADD COLUMN final_fare      NUMERIC; -- locked at booking
ALTER TABLE rides ADD COLUMN surge_reason    TEXT; -- 'demand' | 'time_rule' | null

-- Same 4 columns added to orders table

Redis Keys

surge_config:{regionId}:{rideType}  TTL: 60s
→ Cached JSON of surge_config + surge_time_rules rows for this region/type
→ Populated on first request, invalidated on admin config update
