Dashboards Design — Section 3: Authorization, Error Handling & Observability

Access control, failure modes, logging, and metrics for the Analytics module.

Authorization

Role Scope Access
driver analytics.driver.read Own data only — driver_id extracted from JWT sub
restaurant analytics.restaurant.read Own restaurant only — restaurant_id from JWT sub
admin analytics.admin.read All regions — optional ?region_id= filter

Export endpoints require same scopes. Download URLs are pre-signed (short TTL) — not publicly accessible.

Error Handling

Scenario Handling
No stats yet for period 200 OK with zero-valued summary — never 404
Invalid date range (from > to) 400 Bad Request with clear error message
Export job_id not found 404 Not Found
Export generation fails Job status → failed; client retries via new export request
Kafka consumer duplicate event Deduplicated via processed_events table check before upsert
Consumer DLQ after 5 retries Route to analytics-events-dlq-{region}; alert ops; stats may be temporarily understated

Observability

Log fields
 TraceId, SpanId, RegionId + DriverId or RestaurantId (where applicable), ExportJobId

Key Metrics
analytics_consumer_lag_{region} Alert if consumer falls behind — stats will be stale
analytics_upsert_errors DB upsert failure rate
export_job_duration_ms P95 target < 5s for typical date ranges
export_job_failures Failed CSV/PDF generation rate
